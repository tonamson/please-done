# Chiến lược nâng cấp pd:fix-bug (Detective Orchestrator)

Mục tiêu: Biến `pd:fix-bug` thành một hệ thống điều phối đa Agent (Task Force), tích hợp tinh hoa của `gsd:debug` và sức mạnh MCP của `please-done`.

## 1. Hệ thống Phân phối Model & Tài nguyên (Dynamic Resource Orchestration)

Hệ thống tự động ánh xạ Tier sang model thực tế dựa trên công cụ bạn đang dùng:

| Tier | Vai trò | Gemini CLI (Antigravity) | Claude Code / Cursor |
|------|---------|--------------------------|----------------------|
| **Scout** | Janitor, Doc Specialist | Gemini 3 Flash | Claude 4.5 Haiku |
| **Builder** | Code Detective, Repro | Gemini 3.1 Pro | Claude 4.6 Sonnet |
| **Architect** | Fix Architect (Orchestrator) | Gemini 3.1 Pro / 4 | Claude 4.6 Opus |


**Quy tắc An toàn Tài nguyên:**
- Tối đa **2 Sub-agents** chạy song song.
- **Khóa CPU (Heavy Lock):** Một thời điểm chỉ chạy 1 tác vụ nặng (Indexing/Testing).
- **Hạ cấp thông minh:** Tự động chuyển sang Tuần tự nếu phát hiện RAM yếu hoặc lỗi tài nguyên.

---

## 2. Giao thức Thám tử (The Detective Protocols)

Đảm bảo tính nhất quán và khả năng phục hồi:

- **Resume UI:** Hiển thị danh sách đánh số ID từ `.planning/debug/*.md`. User nhập số để tiếp tục hoặc mô tả mới để tạo session mới.
- **Evidence Format:** Mọi báo cáo thám tử phải tuân thủ:
    - `## ROOT CAUSE FOUND`: Kèm bằng chứng. Orchestrator đưa ra 3 lựa chọn: **(1) Sửa ngay**, **(2) Lên kế hoạch (Plan)**, **(3) Tự sửa**.
    - `## CHECKPOINT REACHED`: Dừng lại hỏi user. Khi user trả lời, spawn **Continuation Agent** (tiếp quản context từ điểm dừng).
    - `## INVESTIGATION INCONCLUSIVE`: Bế tắc. PHẢI có **Elimination Log** (Danh sách các file/logic đã kiểm tra và xác nhận là BÌNH THƯỜNG) để tránh lặp lại giả thuyết sai.

---

## 2.5. Trí nhớ dự án (Project Memory & Regression Detection)

- **Knowledge Recall:** `pd-bug-janitor` PHẢI lục lại `.planning/bugs/` để tìm các lỗi tương tự trong quá khứ.
- **Regression Alert:** Cảnh báo nếu lỗi hiện tại có liên quan đến một bug cũ đã sửa.
- **Double-Check:** Đảm bảo bản sửa mới không phá vỡ các bản sửa cũ dựa trên lịch sử.

---

## 3. Thư viện Agent Thám tử (`commands/pd/agents/`) [ĐÃ HOÀN THÀNH ✅]

Đã triển khai 5 file cấu hình tại `commands/pd/agents/`:
1. `pd-bug-janitor.md` (Scout) - Trí nhớ & Vệ sinh.
2. `pd-code-detective.md` (Builder) - Thám tử hiện trường (FastCode).
3. `pd-doc-specialist.md` (Scout) - Chuyên gia tài liệu (Context7).
4. `pd-repro-engineer.md` (Builder) - Kỹ sư tái hiện (Test case).
5. `pd-fix-architect.md` (Architect) - Tổng tư lệnh & Quyết định.

---

## 4. Workflow Điều phối (The Execution Loop) [ĐANG TRIỂN KHAI ⏳]

1. **Khởi động:** Kiểm tra SESSION cũ hoặc Thu thập triệu chứng (Janitor).
2. **Phân việc:** Spawn Code Detective & Doc Specialist song song.
3. **Tái hiện:** Spawn Repro Engineer để tạo "Red Test".
4. **Phán quyết:** Architect đưa ra kế hoạch sửa lỗi cuối cùng.
5. **Về đích:** Sửa code, xác nhận, cập nhật Truth và commit `[LỖI]`.
