# Command `pd update`

## Purpose
Update or adjust the Plan and Task List when changes come from the User or during execution.

## How It Works
1. **Analyze the change request:** Read User feedback or errors that require a direction change.
2. **Update `PLAN.md`:** Adjust the technical solution, add/remove Truths.
3. **Update `TASKS.md`:** Add new tasks, cancel old tasks, or revise file descriptions.
4. **Re-validate (Plan-Check):** Ensure the revised plan still meets quality standards.

## When to run this command?
- User changes their mind about a feature being worked on.
- A solution from the previous `pd plan` turns out to be infeasible.
- Need to add more Tasks to the current Milestone without scrapping everything.

## Output
- Updated `PLAN.md` and `TASKS.md` files.
- New plan check results report.
