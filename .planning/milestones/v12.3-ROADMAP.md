# Milestone v12.3: Installation & Documentation UX

**Status:** ✅ SHIPPED 2026-04-08

## Overview

Make it possible for a new user to install and understand pd commands in under 5 minutes.

Delivered: a polished installer UX (prompt, progress, error messages, idempotency) and a complete documentation suite (cheatsheet, command reference, workflow overview, getting started guide, README update).

### Phase 146: Installer Reliability

**Goal**: Users see a labeled progress step for every outer install action, and re-running the installer on an up-to-date system exits cleanly.
**Depends on**: Phase 145
**Requirements**: INSTALL-01, INSTALL-03
**Plans**: 3 plans

Plans:
- [x] 146-01: Add checkUpToDate utility to manifest.js, change log.step color to cyan
- [x] 146-02: Add step labels [1/4]–[4/4] and idempotent check to install()
- [x] 146-03: Smoke tests for checkUpToDate and step labels (5 tests)

**Details:**
- `log.step(n, 4, msg)` → renders `[N/4] Message` in cyan
- 4 labeled steps: Backing up → Installing skills → Writing manifest → Syncing agent instructions
- `checkUpToDate(targetDir, VERSION)` → early return if already installed ("Already at vX.Y")
- Upgrade path: logs notice and proceeds normally

### Phase 147: Installer Error Messages

**Goal**: When installation fails, users see an actionable error message with a hint for resolution.
**Depends on**: Phase 145
**Requirements**: INSTALL-02
**Plans**: 2 plans

Plans:
- [x] 147-01: Create error-classifier.js with 4 error categories
- [x] 147-02: Wire classifyError into main().catch() + smoke tests (4 tests)

**Details:**
- `bin/lib/error-classifier.js` — `classifyError(err)` returns `{ category, message, hint }`
- 4 categories: PERMISSION → "sudo chown [path]", MISSING_DEP → "Install via: [URL]", PLATFORM_UNSUPPORTED → "not yet supported", GENERIC → "Run PD_DEBUG=1"
- Wired into `main().catch()` at line 380 of install.js
