**Section 1 -- Format Detection:**
- v1.0 format indicators: YAML frontmatter with `must_haves:` key OR content contains tasks XML tag
- v1.1 format indicators: Markdown Truths table (`| T1 |`) pattern in body + separate TASKS.md file
- Unknown format: return 'unknown', all checks graceful pass

**Section 2 -- CHECK-01: Requirement Coverage (per D-01, D-09, D-10, D-11, D-12):**
- Input: PLAN.md content + requirement IDs from ROADMAP `**Requirements**:` field
- Rule: Each requirement ID must appear as literal text somewhere in PLAN.md content
- If phase has no `Requirements:` field in ROADMAP -> auto PASS (D-10)
- Missing requirement -> BLOCK with message identifying which requirement ID is missing (D-12)
- Matching: regex literal match, escaped for special chars (hyphens in IDs like CHECK-01)

**Section 3 -- CHECK-02: Task Completeness (per D-02, D-05, D-06, D-07, D-08):**
- v1.0 format: Parse task XML elements. Required per task: files tag (non-empty), action or behavior tag (description), verify or done or acceptance_criteria tag (criteria). Missing any BLOCK field -> BLOCK.
- v1.1 format: Parse TASKS.md detail blocks (`## Task N:`). Required metadata fields: Effort, Files, Truths (D-05) -> missing = BLOCK. Required sections: "Mo ta" and "Tieu chi chap nhan" (D-06) must exist and not be empty -> missing/empty = BLOCK. Summary table must have Truths column (D-07) -> missing = BLOCK. Optional metadata: Trang thai, Uu tien, Phu thuoc, Loai (D-08) -> missing = WARN.
- No TASKS.md and v1.0 format -> check v1.0 rules only

**Section 4 -- CHECK-03: Dependency Correctness (per D-03):**
- v1.0: Parse `depends_on:` from frontmatter. Handle formats: `[]` (empty), `[02-01]` (bracket string), `["05-01"]` (quoted bracket string), YAML array (`- 01-01`). For v1.0 single-plan context: graceful PASS (plan-level deps reference other plans not available to checker).
- v1.1: Parse `Phu thuoc` column from TASKS.md summary table + `> Phu thuoc:` metadata. Extract `Task N` references. Validate all referenced tasks exist. Run cycle detection on task dependency graph (Kahn's algorithm).
- Circular dependency -> BLOCK. Invalid reference -> BLOCK.

**Section 5 -- CHECK-04: Truth-Task Coverage (per D-04):**
- v1.0: Parse truths from frontmatter `must_haves: truths:` (nested YAML -- parse raw frontmatter string with regex, not parseFrontmatter which flattens). Tasks don't have explicit Truth refs in v1.0 -> auto PASS for bidirectional check (follows D-10 graceful-skip principle, avoids false positives per D-17).
- v1.1: Parse Truths from PLAN.md Truths table (`| T1 | ... |`). Parse task Truth refs from `> Truths: [T1, T2]` metadata. Check both directions: Truth without any task -> BLOCK. Task without any Truth -> WARN.

Create `bin/lib/plan-checker.js` following the project's pure function module pattern (same style as utils.js: 'use strict', pure functions, module.exports object).

**Imports:**
```javascript
'use strict';
const { parseFrontmatter, extractXmlSection } = require('./utils');
```

**Helper: escapeRegex(str)** -- Escape regex special chars (hyphens, brackets, dots, etc.) so requirement IDs like `CHECK-01` match literally. Implementation: `str.replace(/[.*+?^${}()|[\]\\-]/g, '\\$&')`.

**Helper: parseRequirements(frontmatterObj)** -- Handle parseFrontmatter output variations for `requirements` field:
1. `undefined` -> return `[]`
2. Array (from YAML `- item` notation) -> return as-is
3. String `"[]"` -> return `[]`
4. String `"[LIBR-02, LIBR-03]"` -> strip outer brackets, split on comma, trim each, strip quotes -> `['LIBR-02', 'LIBR-03']`

**Helper: parseDependsOn(frontmatterObj)** -- Same bracket/array/empty handling pattern as parseRequirements for the `depends_on` field.

**Helper: parseMustHavesTruths(rawFrontmatterString)** -- Parse truths from raw frontmatter string (bypass parseFrontmatter which flattens nested keys). Regex to match the `must_haves:\n  truths:\n    - "text"\n    - "text"` block. Extract each quoted string. Return array of `{ id: 'T1', description: '...' }` with auto-incrementing IDs.

**Helper: parseTasksV10(planContent)** -- Extract task XML blocks from v1.0 PLAN.md content. Use regex `/<task[^>]*>([\s\S]*?)<\/task>/g` to find each task block. Within each block, extract: name (from name tags), files (from files tags -- check non-empty), hasDescription (action or behavior tags with content), hasCriteria (verify or done or acceptance_criteria tags with content). Return array of task objects.

**Helper: parseTruthsV11(planContent)** -- Parse Truths table rows from v1.1 PLAN.md. Regex: `/\|\s*(T\d+)\s*\|\s*([^|]+)\s*\|\s*([^|]+)\s*\|/g`. Return array of `{ id, description }`.

**Helper: parseTaskDetailBlocksV11(tasksContent)** -- Split TASKS.md by `## Task N:` heading pattern. For each block extract metadata (Effort, Files, Truths) and section presence (Mo ta, Tieu chi chap nhan). Handle both diacritical and non-diacritical Vietnamese. Return structured task objects.

**Helper: parseSummaryTableV11(tasksContent)** -- Check if TASKS.md summary table header row contains `Truths` column. Return boolean.

**Helper: detectCycles(nodes, edges)** -- Kahn's algorithm for topological sort. Input: array of node IDs + array of `{from, to}` edges. Build in-degree map and adjacency list. Process zero-in-degree nodes. If sorted count less than total nodes, remaining nodes are in cycles. Return `{ hasCycle: boolean, nodesInCycle: string[] }`.

**Helper: findInvalidRefs(validIdSet, edges)** -- For each edge, check if `edge.to` exists in validIdSet. Return array of issue objects for invalid refs.

**detectPlanFormat(planContent):** Check raw frontmatter string for `must_haves:` pattern (indicates v1.0). Also check if content contains the tasks XML opening tag. If either matches, return 'v1.0'. If content matches Truths table pattern `| T1 |`, return 'v1.1'. Otherwise return 'unknown'.

**checkRequirementCoverage(planContent, requirementIds):** If requirementIds empty/null -> return PASS. For each ID, test if it appears literally in planContent (using escaped regex). Missing IDs -> BLOCK issues. Return `{ checkId: 'CHECK-01', status, issues }`.

**checkTaskCompleteness(planContent, tasksContent):** Detect format. For v1.0: parse task XML blocks, validate each has files + description + criteria. For v1.1: parse TASKS.md detail blocks, validate required metadata (Effort/Files/Truths = BLOCK) and sections (Mo ta/Tieu chi chap nhan = BLOCK). Check optional fields (Trang thai/Uu tien/Phu thuoc/Loai = WARN). Check summary table Truths column (BLOCK). Return `{ checkId: 'CHECK-02', status, issues }`.

**checkDependencyCorrectness(planContent, tasksContent):** For v1.0: return PASS (single-plan context, no task-level deps). For v1.1: parse task dependencies from TASKS.md, build graph, run findInvalidRefs + detectCycles. Return `{ checkId: 'CHECK-03', status, issues }`.

**checkTruthTaskCoverage(planContent, tasksContent):** For v1.0/unknown: return PASS (no Truth-Task mapping available). For v1.1: parse Truths from PLAN.md table + task Truth refs from TASKS.md. Check both directions: Truth not in any task -> BLOCK, Task Truth not in Truths table -> WARN. Return `{ checkId: 'CHECK-04', status, issues }`.

**runAllChecks({ planContent, tasksContent, requirementIds }):** Run all 4 checks. Aggregate: overall = 'block' if any block, 'warn' if any warn, else 'pass'. Return `{ overall, checks }`.

**module.exports:** Export all 6 main functions + all helpers (for unit testing): detectPlanFormat, checkRequirementCoverage, checkTaskCompleteness, checkDependencyCorrectness, checkTruthTaskCoverage, runAllChecks, escapeRegex, parseRequirements, parseDependsOn, parseMustHavesTruths, parseTasksV10, parseTruthsV11, parseTaskDetailBlocksV11, detectCycles, findInvalidRefs.
