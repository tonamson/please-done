# v12.3 Requirements Archive: Installation & Documentation UX

**Milestone:** v12.3
**Status:** ✅ SHIPPED 2026-04-08
**Goal:** Make it possible for a new user to install and understand pd commands in under 5 minutes.
**Defined:** 2026-04-07

---

## Requirements

### Installation UX

- [x] **INSTALL-01**: User sees clear progress steps during installation
  - Each step has a label (e.g., "Installing Claude Code skills…")
  - Per-step success/failure indicator (✓ / ✗)
  - No silent operations — every action is announced
  - **Outcome:** `log.step(n, 4, msg)` renders `[N/4] Message` in cyan; 4 labeled steps in install.js

- [x] **INSTALL-02**: When installation fails, user knows the reason and how to fix it
  - Actionable error messages replace raw stack traces
  - Each error category (missing deps, permission, platform) has a specific fix hint
  - Exit code is non-zero on failure
  - **Outcome:** `bin/lib/error-classifier.js` with 4 categories + hints; wired into `main().catch()`

- [x] **INSTALL-03**: Running the installer again does not break an existing install
  - Already-installed files show "already installed" status, not errors
  - Re-run updates only changed files
  - Version upgrade path is handled gracefully
  - **Outcome:** `checkUpToDate()` returns early with "Already at vX.Y" message

- [x] **INSTALL-04**: Interactive platform selector is intuitive
  - Numbered choice fallback for non-TTY
  - Each platform shows a one-line description
  - Selected platform is confirmed before proceeding
  - **Outcome:** `bin/lib/prompt.js` with TTY/non-TTY modes; 7 platform descriptions; confirmation line

### Documentation

- [x] **DOCS-01**: README quick start reflects current command set
  - Commands match v12.3 feature set
  - New commands visible in Skills Reference
  - Version badge updated to current version
  - **Outcome:** README updated to v12.3.0; 20 skills in 5 categories; sync-version/stats/health/discover added

- [x] **DOCS-02**: docs/cheatsheet.md covers all 20 current commands
  - Adds pd:stats, pd:health, pd:discover, pd:sync-version sections
  - Each command has usage + one-line description
  - **Outcome:** 20 commands in 5 categories (Project/Planning/Execution/Debug/Utility); review fixes applied

- [x] **DOCS-03**: docs/COMMAND_REFERENCE.md is rewritten as concise per-command reference
  - Each command: purpose (1 sentence) + syntax + real example
  - Inline — no links to missing commands/ subdirectory
  - **Outcome:** 123 lines, 20 command blocks, Purpose/Syntax/Example format

- [x] **DOCS-04**: docs/WORKFLOW_OVERVIEW.md is rewritten — short and flow-focused
  - Maximum 60 lines
  - Answers: "when do I use which command?"
  - Simple flow diagram showing the lifecycle
  - **Outcome:** 34 lines; Mermaid `flowchart LR`; 8-command quick-reference table

- [x] **DOCS-05**: docs/GETTING_STARTED.md exists and guides a new user to first phase completion
  - Step-by-step: install → onboard → first plan → first phase done
  - Estimated time for each step
  - Common pitfalls called out inline
  - **Outcome:** 75 lines; 5 steps with time estimates; 2 pitfall callouts; links to WORKFLOW_OVERVIEW + COMMAND_REFERENCE
