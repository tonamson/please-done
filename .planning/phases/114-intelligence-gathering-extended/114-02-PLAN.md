  <action>
Create bin/lib/auth-analyzer.js implementing the AuthAnalyzer class with these methods:

1. Constructor accepting options with cache parameter

2. analyze(projectPath, targetInfo, options) - main entry returning Promise with:
   - authPatterns: array of detected middleware patterns
   - jwtAnalysis: JWT implementation vulnerabilities
   - hardcodedCredentials: credential exposures found
   - coverageMatrix: auth coverage analysis

3. detectAuthMiddleware(ast, filePath) - AST traversal for:
   - Express: app.use(authMiddleware) patterns
   - JWT: jwt.verify() calls (per D-07, D-10)
   - Passport: passport.authenticate() calls
   - Session: session middleware detection
   - OAuth: OAuth provider patterns
   Returns array of {type, file, line, pattern}

4. analyzeJwtImplementation(ast, filePath) - checks for:
   - jwt.verify() without algorithms option (CVE-2024-54150 pattern)
   - jwt.sign() with sensitive payload data
   - Algorithm confusion vulnerabilities
   Returns array of vulnerability objects with severity

5. findHardcodedCredentials(ast, filePath) - AST traversal for:
   - VariableDeclarator with credential pattern names and StringLiteral values
   - LogicalExpression with fallback credentials (process.env.SECRET || 'fallback')
   - Filters out test files (*.test.js, *.spec.js)
   Returns array of credential findings

6. generateAuthCoverageMatrix(routes, authPatterns) - per D-12:
   - Input: routes from target-enumerator.js, authPatterns from detection
   - Output: {protected: [], unprotected: [], bypassCandidates: [], summary: {}}
   - bypassCandidates identified for sensitive routes without auth

Export: { AuthAnalyzer, generateAuthCoverageMatrix }
  </action>
