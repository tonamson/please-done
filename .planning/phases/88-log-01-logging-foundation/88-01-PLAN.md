# Phase 88: LOG-01 — Agent Error Logging Foundation

## Goal

Create structured logging infrastructure and log-writer utility for agent error tracking.

## Requirements

- **LOG-01**: Agent error structured logging — JSONL at `.planning/logs/agent-errors.jsonl` (timestamp, level, phase, step, agent, error, context)

## Success Criteria

1. `.planning/logs/` directory created with `.gitignore` rules
2. `bin/lib/log-writer.js` pure function for JSONL logging
3. Log format: `{timestamp, level, phase, step, agent, error, context}`
4. Unit tests for log-writer with 90%+ coverage

## Effort Estimate

**Size:** Small  
**Complexity:** Low  
**Risk:** Minimal (new infrastructure, no breaking changes)

## Plan

### Task 1: Create log directory and .gitignore

- Create `.planning/logs/` directory
- Add to root `.gitignore`: `.planning/logs/*.jsonl`
- Verify directory structure

### Task 2: Implement log-writer.js utility

- Create `bin/lib/log-writer.js`
- Implement `writeLog(entry)` pure function
- Format: JSONL with fields: timestamp (ISO), level, phase, step, agent, error, context (optional)
- Handle file I/O errors gracefully (console fallback)
- Add JSDoc comments

### Task 3: Write unit tests

- Create `test/log-writer.test.js`
- Test successful log write
- Test error handling fallback
- Test JSONL format validation
- Achieve 90%+ coverage

### Task 4: Integration verification

- Run full test suite: `node --test test/log-writer.test.js`
- Verify no regressions in existing tests
- Check log file creation and format
