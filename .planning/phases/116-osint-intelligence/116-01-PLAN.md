# Phase 116 Plan 01: OSINT Core Libraries

## Objective

Create foundational OSINT libraries for external reconnaissance without direct target interaction.

## Wave 1: OSINT Library Foundation

### Task 1: Google Dorks Generator
```yaml
action: |
  Create bin/lib/google-dorks.js with categorized Google Dork query generation:

  1. Site enumeration dorks:
     - site:target.com inurl:admin
     - site:target.com inurl:login
     - site:target.com inurl:api

  2. Exposed file dorks:
     - site:target.com ext:sql OR ext:bak OR ext:config
     - site:target.com ext:log OR ext:xml OR ext:json
     - site:target.com filetype:env OR filetype:ini

  3. Sensitive info dorks:
     - site:target.com intitle:"index of"
     - site:target.com intitle:"Directory Listing"
     - site:target.com "password" OR "secret" OR "key"

  4. Error disclosure dorks:
     - site:target.com "error" OR "warning" OR "exception"
     - site:target.com "stack trace" OR "debug"

  Export class GoogleDorks with methods:
  - generate(target, categories) - returns array of dork objects
  - getCategories() - returns available categories
  - exportForTool(tool, dorks) - formats for specific tool (browser CLI, etc)

read_first:
  - bin/lib/recon-cache.js
  - docs/pentest/OSINT.md
  - docs/mitre/T1593.002.md

acceptance_criteria:
  - bin/lib/google-dorks.js exists and exports GoogleDorks class
  - generate() returns array with at least 20 dork queries per target
  - Each dork object has: query, category, description, mitre_technique
  - Unit tests pass: test/lib/google-dorks.test.js
  - Can generate queries for "example.com" covering all categories
```

### Task 2: Certificate Transparency Scanner
```yaml
action: |
  Create bin/lib/ct-scanner.js for CT log subdomain discovery:

  1. Implement CrtShProvider class:
     - query(domain) - fetches subdomains from crt.sh
     - Handles rate limiting (1 req/sec default)
     - Returns parsed subdomain list with certificate details

  2. Implement CensysProvider class (optional API key):
     - query(domain) - fetches from Censys API v2
     - Requires CENSYS_API_ID and CENSYS_SECRET env vars
     - Returns enhanced cert data with organization info

  3. Implement CertSpotterProvider class (optional API key):
     - query(domain) - fetches from CertSpotter API
     - Free tier: 100 queries/hour

  4. Create CtScanner coordinator:
     - scan(domain, options) - aggregates all providers
     - Deduplicates subdomains across sources
     - Caches results in recon-cache
     - Respects rate limits with exponential backoff

  Export classes: CrtShProvider, CensysProvider, CertSpotterProvider, CtScanner

read_first:
  - bin/lib/recon-cache.js
  - docs/pentest/OSINT.md
  - docs/mitre/T1596.003.md

acceptance_criteria:
  - bin/lib/ct-scanner.js exists with all exported classes
  - CrtShProvider.query() returns array of {subdomain, issuer, not_after}
  - Deduplication removes exact subdomain duplicates
  - Rate limiting prevents >1 request/sec to crt.sh
  - Unit tests: test/lib/ct-scanner.test.js (mock HTTP)
  - Can discover subdomains for "google.com" (test with mocks)
```

### Task 3: Repository Secret Detector
```yaml
action: |
  Create bin/lib/secret-detector.js for detecting secrets in code:

  1. Implement pattern matching for:
     - AWS: AKIA[0-9A-Z]{16}, ASIA[0-9A-Z]{16}
     - GitHub: ghp_[a-zA-Z0-9]{36}, gho_[a-zA-Z0-9]{36}
     - Slack: xox[baprs]-[0-9]{10,13}-[0-9]{10,13}-[a-zA-Z0-9]{24}
     - Generic API keys: (?i)(api[_-]?key|apikey)[\s]*[=:][\s]*['"][a-z0-9]{16,}
     - Database URLs: (?i)(postgres|mysql|mongodb)://[^:]+:[^@]+@
     - JWT tokens: eyJ[a-zA-Z0-9_-]*\.eyJ[a-zA-Z0-9_-]*\.[a-zA-Z0-9_-]*
     - Private keys: -----BEGIN (RSA |EC |OPENSSH )?PRIVATE KEY-----
     - Password patterns: (?i)password[\s]*[=:][\s]*['"][^'"]{8,}

  2. Implement SecretDetector class:
     - scan(text) - scans arbitrary text, returns matches
     - scanFile(filePath) - reads and scans file
     - scanGitHubRepo(owner, repo, options) - uses GitHub API
     - validateFinding(finding) - reduces false positives

  3. Implement GitHub API integration:
     - Rate limit handling (5000 req/hour authenticated)
     - Search code API for patterns
     - Contents API for file retrieval
     - Requires GITHUB_TOKEN env var

read_first:
  - bin/lib/recon-cache.js
  - docs/pentest/OSINT.md
  - docs/mitre/T1552.md
  - bin/lib/validation-patterns.js

acceptance_criteria:
  - bin/lib/secret-detector.js exists and exports SecretDetector
  - scan() returns array of {pattern, match, line, confidence, rule_id}
  - Detects at least 5 common secret types (AWS, GitHub, Slack, JWT, DB URL)
  - False positive rate <10% (validateFinding filters common patterns)
  - Unit tests: test/lib/secret-detector.test.js
  - GitHub API integration handles rate limits gracefully
```

### Task 4: Subdomain OSINT Aggregator
```yaml
action: |
  Create bin/lib/subdomain-osint.js combining all OSINT sources:

  1. Implement SubdomainOsint class:
     - constructor(cache, options) - accepts recon-cache instance
     - enumerate(domain, sources) - discovers subdomains from multiple OSINT sources
     - getSources() - returns available sources and their status

  2. Integrate sources:
     - CT logs (via ct-scanner.js)
     - Passive DNS (optional integration point)
     - Certificate fingerprints
     - DNS TXT records for SPF/DMARC subdomain hints

  3. Implement result aggregation:
     - Merge subdomains from all sources
     - Track source provenance for each subdomain
     - Score confidence based on source count
     - Remove wildcards (*.example.com -> extract actual)

  4. Implement output formatting:
     - toJSON() - structured data with metadata
     - toCSV() - flat list for spreadsheet import
     - toTable() - CLI display format

read_first:
  - bin/lib/ct-scanner.js
  - bin/lib/recon-cache.js
  - docs/pentest/OSINT.md

acceptance_criteria:
  - bin/lib/subdomain-osint.js exports SubdomainOsint class
  - enumerate() returns array of {subdomain, sources, confidence, first_seen}
  - Aggregates from at least 2 CT log sources
  - Confidence scoring: 1 source=low, 2=medium, 3+=high
  - Deduplicates subdomains (case-insensitive)
  - Unit tests: test/lib/subdomain-osint.test.js
```
