<decisions>
## Implementation Decisions

### Base converter architecture
- **D-01:** Template method pattern with config object — a single `convertSkill(content, platformConfig)` function that accepts platform-specific hooks
- **D-02:** Platform config object contains: `toolMap`, `pathReplace`, `configGenerator`, `outputStructure`, `mcpToolConverter`, and any platform-specific post-processing hooks
- **D-03:** Shared conversion pipeline order: `parseFrontmatter → inlineGuardRefs → inlineWorkflow → classifyRefs → pathReplace → toolMap → mcpToolConvert → rebuildFrontmatter`
- **D-04:** Each platform converter file reduces to: config object definition + platform-specific functions (configGenerator, merge/strip helpers)
- **D-05:** Keep pure function style — no classes, matches existing codebase conventions

### Shared logic extraction
- **D-06:** These operations move into base converter (currently duplicated across 4 files):
  - Frontmatter parse/rebuild with field selection
  - Path replacement (`~/.claude/` → platform path)
  - Tool name mapping with word-boundary regex
  - MCP tool reference conversion
  - Workflow inlining + guard ref expansion (already in utils.js, base converter orchestrates the call)
- **D-07:** Platform-specific logic that stays in individual configs:
  - Codex: `$ARGUMENTS` → `{{GSD_ARGS}}`, skill adapter XML generation, TOML config
  - Copilot: instructions.md merging with HTML markers, MCP namespace conversion (mcp__X__Y → vendor/X/Y)
  - Gemini: `${VAR}` → `$VAR` escaping, JSON MCP config
  - OpenCode: flat file naming (`flattenName()`), minimal field stripping

### Error handling scope
- **D-08:** Scope includes converters, manifest.js, AND installers — entire install pipeline, not just converters
- **D-09:** Silent `catch {}` and `catch { /* ignore */ }` blocks must either: (a) log a warning with context, or (b) re-throw with descriptive message
- **D-10:** Error classification:
  - **Hard errors** (re-throw): file write failures, converter output corruption, config generation failures
  - **Soft warnings** (log + continue): broken symlinks during scan, legacy file cleanup, optional config merge
- **D-11:** No error accumulation pattern — fail fast on hard errors, log+continue on soft warnings
- **D-12:** Specific silent catches to fix:
  - `manifest.js:31` — broken symlink: log warning with path
  - `manifest.js:72,90,180` — JSON parse: log warning with file path and error
  - `installers/claude.js:62-68` — uv install chain: log each fallback attempt
  - `installers/claude.js:97-100` — venv creation: log fallback
  - `installers/gemini.js:109` — settings.json parse: already logs, acceptable
  - `installers/gemini.js:148` — silent catch: log warning

### Verification strategy
- **D-13:** Snapshot testing — capture converter output for all skill files BEFORE refactoring, compare AFTER
- **D-14:** Snapshot test covers all 4 converters × all 12 skills = 48 output comparisons
- **D-15:** Snapshots stored in `tests/snapshots/` directory, generated by a one-time script
- **D-16:** Snapshot comparison ignores whitespace-only differences (trailing newlines, indentation normalization)
- **D-17:** Run existing `smoke-integrity.test.js` + new snapshot tests as verification gate

### Claude's Discretion
- Base converter file location (new `bin/lib/converters/base.js` vs extend `utils.js`)
- Exact config object shape and default values
- Snapshot generation script implementation
- Whether to split large platform configs into separate helper files
- Test file organization for snapshot tests

</decisions>

## Existing Code Insights

### Reusable Assets
- `parseFrontmatter()` / `buildFrontmatter()` in utils.js: Already shared, base converter will orchestrate calls
- `inlineWorkflow()` in utils.js: Already shared, handles required/optional ref separation (Phase 4)
- `inlineGuardRefs()` in utils.js: Already shared, guard expansion

- Pure function style throughout — no classes anywhere in codebase
- `(required)` / `(optional)` tagging in execution_context — Phase 1/4 convention
- Tool mapping uses word-boundary regex: `new RegExp(\`\\b${claude}\\b(?!\\()\`, 'g')`
- Idempotent config merging with HTML/XML markers (Codex, Copilot)
- `log.info()`, `log.warn()`, `log.success()` — existing logging utilities in utils.js

### Integration Points
- All 4 converters export `convertSkill()` — base converter must maintain this API
- `bin/install.js` imports converters dynamically — must keep same module interface
- `manifest.js` called by all installers — error changes affect entire pipeline
- `smoke-integrity.test.js` — must remain green throughout refactoring
