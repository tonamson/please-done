<objective>
Create a version synchronization tool (pd:sync-version) that keeps version numbers consistent across all project files. The tool reads the version from package.json (source of truth) and syncs it to README.md badge URLs, version text lines, and doc files with `<!-- Source version: X.Y.Z -->` comment headers. Includes --check mode for validation-only and integration with the complete-milestone workflow.

Purpose: Eliminate manual version updates that cause drift between package.json, README badges, and doc headers. Currently 30+ version references across 28+ files that must be updated manually on each release.

Output: Pure-function library (bin/lib/version-sync.js), comprehensive tests (test/version-sync.test.js), skill definition (commands/pd/sync-version.md), and complete-milestone workflow integration.
</objective>

    1. `extractPackageVersion(content)` — JSON.parse content, return `pkg.version` or `null`. Guard: null/empty/non-string returns null, JSON.parse error returns null.

    2. `extractBadgeVersion(content)` — Regex: `/version-(\d+\.\d+\.\d+)-blue/`, return match[1] or null. This matches both README.md and README.vi.md badge URLs.

    3. `extractTextVersion(content)` — Regex: `/\*\*Current version: v?(\d+\.\d+\.\d+)\*\*/`, return match[1] or null. The `v?` handles both `v4.0.0` and `4.0.0`.

    4. `extractDocVersion(content)` — Regex: `/<!-- Source version: (\d+\.\d+\.\d+) -->/`, return match[1] or null.

    5. `replaceBadgeVersion(content, newVersion)` — Replace `version-(\d+\.\d+\.\d+)-blue` with `version-${newVersion}-blue`. Guard: return content unchanged if pattern not found.

    6. `replaceTextVersion(content, newVersion)` — Replace `(\*\*Current version: v?)\d+\.\d+\.\d+(\*\*)` with `$1${newVersion}$2`. This preserves the `v` prefix if present.

    7. `replaceDocVersion(content, newVersion)` — Replace `<!-- Source version: \d+\.\d+\.\d+ -->` with `<!-- Source version: ${newVersion} -->`. Guard: return content unchanged if pattern not found.

    8. `compareVersions(expectedVersion, fileResults)` — Takes expected version string and array of `{ file, extractor, content }` objects. For each entry: call extractor(content) → compare to expectedVersion → return `{ file, current, expected, status: 'match'|'mismatch'|'no_version' }`. Return full results array.

Status display: match → `✓ OK`, mismatch → `✗ MISMATCH`, no_version → `— NO VERSION`. Include header with expected version, summary line with total/mismatch counts.
