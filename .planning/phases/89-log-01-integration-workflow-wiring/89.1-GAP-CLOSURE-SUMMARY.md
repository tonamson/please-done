## How It Works

### Enhanced Error Handlers (Critical Skills)

Each critical skill gets a specialized error handler that enriches context:

```javascript
// In fix-bug.md
const { createFixBugErrorHandler } = require('../../../bin/lib/enhanced-error-handler');
const errorHandler = createFixBugErrorHandler('$CURRENT_PHASE', {
  bugDescription: typeof $ARGUMENTS !== 'undefined' ? $ARGUMENTS : 'unknown',
  sessionId: new Date().toISOString(),
  currentStep: 'initialization'
});
module.exports = { errorHandler };
```

### Basic Error Handlers (Remaining Skills)

Basic skills get a standard error handler:

```javascript
// In init.md
const { createBasicErrorHandler } = require('../../../bin/lib/basic-error-handler');
const errorHandler = createBasicErrorHandler('pd:init', '$CURRENT_PHASE', {
  operation: 'init'
});
module.exports = { errorHandler };
```

### Error Handling Flow

1. When a skill encounters an error, it calls `errorHandler.handle(error, context)`
2. The error handler logs structured data to `.planning/logs/agent-errors.jsonl`
3. The error is re-thrown to maintain existing behavior
4. The log-reader can display errors via what-next
