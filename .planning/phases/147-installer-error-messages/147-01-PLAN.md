# Plan 147-01: Create error-classifier.js

## Goal
Create `bin/lib/error-classifier.js` — a zero-dependency pure function that inspects any thrown `Error` and returns `{ category, message, hint }` for clean terminal output. No I/O, no side effects.

## Context
Per D-01 through D-06 and D-12 from `147-CONTEXT.md`:
- 4 categories: `MISSING_DEP`, `PERMISSION`, `PLATFORM_UNSUPPORTED`, `GENERIC`
- Detection priority: `err.code` first, then `err.message` regex
- Hints are fixed strings per category (with path interpolation for PERMISSION)
- Export: `module.exports = { classifyError }` only
- Zero npm dependencies — Node.js built-ins only (no `require()` needed here at all)

Reference `bin/lib/manifest.js` for the module structure pattern (`'use strict'`, JSDoc, `module.exports = {...}` at bottom).

## Tasks

### Task 1: Create bin/lib/error-classifier.js

**File(s)**: `bin/lib/error-classifier.js`

**Action**:

Create the file with `'use strict'` at the top. Implement `classifyError(err)` exactly as specified by D-01 through D-06:

**Detection logic (evaluate in this order):**

1. `err.code === 'EACCES' || err.code === 'EPERM'`  
   → category: `'PERMISSION'`  
   → message: `err.message` (the raw fs error text is already descriptive)  
   → hint: if `err.path` exists → `` `Fix: sudo chown $(whoami) ${err.path}` ``; else → `'Check file permissions in your home directory'`

2. `err.code === 'MODULE_NOT_FOUND'`  
   → category: `'PLATFORM_UNSUPPORTED'`  
   → message: `err.message`  
   → hint: `'This platform is not yet supported. See: https://github.com/tonamson/please-done for supported platforms'`

3. `/not installed|not found|requires|missing/i.test(err.message || '')`  
   → category: `'MISSING_DEP'`  
   → message: `err.message`  
   → hint: extract URL from `err.message` using `/https?:\/\/\S+/` — if found use it as hint; else `'Install the missing dependency and re-run the installer'`

4. (fallback) → category: `'GENERIC'`, message: `err.message || String(err)`, hint: `'Run with PD_DEBUG=1 for more details'`

**Important**: The function must NOT call `log.*`, `console.*`, or `process.*`. It is a pure classifier — callers handle output.

Export at the bottom: `module.exports = { classifyError };`

**Full file structure to produce:**

```js
'use strict';

/**
 * Classify an installer error into a structured result for clean terminal output.
 *
 * @param {Error} err
 * @returns {{ category: string, message: string, hint: string }}
 */
function classifyError(err) {
  const code = err.code || '';
  const msg  = err.message || String(err);

  // PERMISSION — filesystem access denied
  if (code === 'EACCES' || code === 'EPERM') {
    const hint = err.path
      ? `Fix: sudo chown $(whoami) ${err.path}`
      : 'Check file permissions in your home directory';
    return { category: 'PERMISSION', message: msg, hint };
  }

  // PLATFORM_UNSUPPORTED — runtime installer not found
  if (code === 'MODULE_NOT_FOUND') {
    return {
      category: 'PLATFORM_UNSUPPORTED',
      message: msg,
      hint: 'This platform is not yet supported. See: https://github.com/tonamson/please-done for supported platforms',
    };
  }

  // MISSING_DEP — dependency not present (detected via message text)
  if (/not installed|not found|requires|missing/i.test(msg)) {
    const urlMatch = msg.match(/https?:\/\/\S+/);
    const hint = urlMatch
      ? urlMatch[0]
      : 'Install the missing dependency and re-run the installer';
    return { category: 'MISSING_DEP', message: msg, hint };
  }

  // GENERIC — catch-all
  return {
    category: 'GENERIC',
    message: msg,
    hint: 'Run with PD_DEBUG=1 for more details',
  };
}

module.exports = { classifyError };
```

**Expected result**: `node -e "const {classifyError}=require('./bin/lib/error-classifier'); console.log(classifyError({code:'EACCES',path:'/tmp/x'}))"` prints `{ category: 'PERMISSION', message: ..., hint: 'Fix: sudo chown $(whoami) /tmp/x' }` without error.
