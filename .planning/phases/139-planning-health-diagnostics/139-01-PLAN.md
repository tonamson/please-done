  <action>
    Create bin/lib/health-checker.js as a CommonJS pure-function library following the exact pattern of stats-collector.js:

    1. Start with `'use strict';` — no fs/path imports needed (pure functions only)

    2. Define SEVERITY_LEVEL enum:
    ```javascript
    const SEVERITY_LEVEL = {
      CRITICAL: 'critical',
      WARNING: 'warning',
      INFO: 'info',
    };
    ```

    3. Define checkMissingFiles(phaseDirs, completedPhases):
       - Parameters: phaseDirs = array of { name: string, files: string[] }, completedPhases = array of phase numbers
       - For each completed phase (phase number in completedPhases), check if SUMMARY.md and VERIFICATION.md exist in its files list
       - Return issues array with severity WARNING for missing files, category 'missing_files'
       - Fix suggestions: 'Run /gsd-validate-phase {N}' for missing VERIFICATION.md, 'Re-run Phase {N} execution' for missing SUMMARY.md

    4. Define checkStateMdStructure(content):
       - Parse YAML frontmatter from content using regex (same pattern as stats-collector.js parseStateProgress)
       - Required top-level fields per D-17: gsd_state_version (string), milestone (string, non-empty), status (string, non-empty)
       - Required nested fields under progress: total_phases (number), completed_phases (number), total_plans (number), completed_plans (number), percent (number)
       - Validate types per D-18: gsd_state_version must be string, progress.percent must be numeric, status must be non-empty
       - Missing top-level required fields → severity CRITICAL, category 'state_schema'
       - Missing progress sub-fields → severity CRITICAL
       - Invalid field types → severity WARNING
       - Fix suggestion: 'Edit .planning/STATE.md to add missing {field_name} field'

    5. Define checkOrphanedDirs(phaseDirs, roadmapPhases):
       - Parameters: phaseDirs = array of directory name strings (e.g., ['137-workflow-command-merge', '138-project-statistics-command']), roadmapPhases = array of phase numbers from roadmap
       - Extract phase number from directory name (regex: /^(\d+)-/)
       - If directory's phase number not in roadmapPhases → severity INFO, category 'orphaned_dirs'
       - Fix suggestion: 'Delete .planning/phases/{dirName}/ or add to ROADMAP.md'

    6. Define runAllChecks(planningDir, roadmapContent, stateContent):
       - Orchestrator function that calls the three checks
       - For checkMissingFiles: scan planningDir/phases/ for subdirectories, read each dir's file list, get completed phases from roadmapContent
       - For checkStateMdStructure: pass stateContent directly
       - For checkOrphanedDirs: get dir names and roadmap phase numbers
       - Return combined array of all issues sorted by severity (critical first, then warning, then info)

    7. Define formatHealthReport(issues):
       - If issues empty: return "All checks passed ✓" string
       - Otherwise: build boxed table output with summary line at top
       - Summary: "Health check: {N} issues found ({Y} critical, {Z} warning, {W} info)"
       - Group issues by category, render each group as a section with ╔═╗║╚╝ borders
       - Each issue row: severity badge (colored), location, issue description, fix command
       - Use padRight for column alignment (same as stats-collector.js pattern)
       - Color severity: critical=red, warning=yellow, info=cyan (using COLORS from utils.js conceptually — formatHealthReport returns plain string, caller applies colors)

    8. Export all functions:
    ```javascript
    module.exports = {
      SEVERITY_LEVEL,
      checkMissingFiles,
      checkStateMdStructure,
      checkOrphanedDirs,
      runAllChecks,
      formatHealthReport,
    };
    ```
