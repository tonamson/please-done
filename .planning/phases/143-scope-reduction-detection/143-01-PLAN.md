<background>
**Phase 139 establishes the pattern this module must follow:**
- `bin/lib/health-checker.js` = zero fs imports, pure functions, content passed as params
- Issue shape: `{ severity, category, location, issue, fix }`
- `SEVERITY_LEVEL` enum for consistency across diagnostic modules
- `formatHealthReport` = boxed table with ╔═╗/║/╚═╝ borders, 70-char width

**Why a separate file (D-01):** health-checker.js covers structural diagnostics (missing files, STATE.md schema, orphaned dirs). scope-checker.js covers *semantic* diagnostics — did the execution deliver what the plan promised? Keeping them separate maintains single-responsibility and allows independent testing.

**What scope reduction means (L-07):**
- A PLAN.md declares `requirements: [L-07]` and `must_haves.artifacts[{path: "bin/lib/foo.js"}]`
- The corresponding SUMMARY.md must mention `L-07` somewhere in its text AND reference `bin/lib/foo.js` in backticks
- If either is absent → scope was silently reduced → WARNING issue

**pd:health.md update (D-07):** The skill file needs one new documented step showing callers how to also run `checkScopeReductions` alongside `runAllChecks`. This is a documentation addition — no code change to health-checker.js itself.
</background>

## Task 2 — Implement scope-checker.js (GREEN phase)

**Files:** `bin/lib/scope-checker.js`

**Objective:** Create the library making all 25 tests pass. Zero fs imports, pure functions, content passed as parameters.

**File header (must match health-checker.js module documentation pattern):**

```javascript
/**
 * Scope Checker Module — Pure functions for detecting scope reduction between PLAN.md and SUMMARY.md.
 *
 * Pure functions: does NOT read files, zero fs imports, NO side effects.
 * Content passed via parameters, returns issue arrays.
 *
 * - parsePlanFile: extract requirements[], truths[], artifacts[], phase from PLAN.md content
 * - parseSummaryFile: extract mentionedReqs[], deliveredPaths[], status, phase from SUMMARY.md
 * - detectReductions: compare plan vs summary, return { issues, droppedReqs, droppedArtifacts }
 * - checkScopeReductions: orchestrator accepting [{planContent, summaryContent, label}] pairs
 * - formatScopeReport: render issues as boxed table; "No scope reductions detected ✓" if empty
 */

'use strict';

const yaml = require('js-yaml');
```

**Implementation notes:**

`parsePlanFile(content)`:
- Use `content.match(/^---\n([\s\S]*?)\n---/)` to extract frontmatter block
- Use `yaml.load(fmMatch[1])` to parse; wrap in try/catch, default to `{}`
- `requirements`: `fm.requirements || []` — ensure it is always an array
- `truths`: `fm.must_haves && fm.must_haves.truths ? fm.must_haves.truths : []`
- `artifacts`: `fm.must_haves && fm.must_haves.artifacts ? fm.must_haves.artifacts : []`
- `phase`: `fm.phase || null`
- Return `{ requirements: [], truths: [], artifacts: [], phase: null }` for empty/null input

`parseSummaryFile(content)`:
- Extract frontmatter via same regex; `yaml.load` for `status` and `phase`
- `mentionedReqs`: `[...content.matchAll(/L-\d+/g)].map(m => m[0])` deduplicated via `[...new Set(...)]`
- `deliveredPaths`: extract all backtick tokens: `` content.match(/`([^`]+)`/g) `` → strip backticks → filter to tokens containing `/` or `.` AND at least one letter before the dot/slash (avoids `` `true` ``, `` `npm test` ``)
  - Simple heuristic: token must match `/[a-zA-Z].*[/.]/` 
- Return `{ status: '', phase: null, mentionedReqs: [], deliveredPaths: [] }` for empty/null

`detectReductions(plan, summary)`:
- `droppedReqs = plan.requirements.filter(r => !summary.mentionedReqs.includes(r))`
- `droppedArtifacts = plan.artifacts.filter(a => !summary.deliveredPaths.includes(a.path))`
- For each droppedReq: push issue `{ severity: SEVERITY_LEVEL.WARNING, category: 'scope_reduction', location: locationStr, issue: \`Dropped requirement: ${req} (in plan, absent from summary)\`, fix: \`Review SUMMARY.md and verify ${req} was intentionally deferred\` }`
- For each droppedArtifact: push issue `{ severity: SEVERITY_LEVEL.WARNING, category: 'scope_reduction', location: locationStr, issue: \`Dropped artifact: ${a.path} (in must_haves.artifacts, not found in summary)\`, fix: \`Verify ${a.path} was delivered or intentionally deferred\` }`
- `locationStr`: `plan.phase ? \`Phase \${plan.phase}\` : 'unknown phase'`

`checkScopeReductions(pairs)`:
- If not array or empty: return `[]`
- For each pair: call `parsePlanFile(pair.planContent)`, `parseSummaryFile(pair.summaryContent)`
  - If `plan.phase` is null and `pair.label`: set `plan.phase = pair.label` before calling detectReductions
  - Call `detectReductions(plan, summary)`, collect `.issues`
- Return flat array: `pairs.flatMap(...)` or accumulate and concat

`formatScopeReport(issues)`:
- Empty → return `'No scope reductions detected ✓'`
- Non-empty → replicate `formatHealthReport` pattern from health-checker.js:
  - **Before/after summary line:** `Scope check: Plan declared: N req, M artifacts / Summary delivered: P req, Q artifacts`
    where N = total unique req IDs across all pairs, P = those found in summaries, etc.
    (Simplification: derive counts from the issues passed in — each issue carries `location` with the label)
  - Boxed table section with label `'Scope Reduction'` (category: scope_reduction)
  - Each issue: `[WARNING] {issue.issue}`, `    Location: {issue.location}`, `    Fix: {issue.fix}`
  - Use `padRight` helper (copy verbatim from health-checker.js)

`module.exports`:
```javascript
module.exports = {
  parsePlanFile,
  parseSummaryFile,
  detectReductions,
  checkScopeReductions,
  formatScopeReport,
};
```
