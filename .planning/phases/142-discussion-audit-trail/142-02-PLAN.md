# Plan 142-02: Auto-Capture Hook in discuss-phase.md

## Goal

Close verification gap: inject a `capture_context` block into the `write_context` step of
`discuss-phase.md` so that every completed discuss-phase session automatically writes a
distilled summary to `.planning/contexts/{padded_phase}-{YYYY-MM-DD}.md`.

This feeds the `pd:audit` library (audit-trail.js) with data so users can list, search, and
resume from stored context summaries.

## Why

Phase 142 delivered the read side (library + skill) but not the write side (auto-capture).
Without context files, pd:audit always shows "No contexts found" — the feature is inert.

## Context File Format

Must match what `parseContextFile()` in `bin/lib/audit-trail.js` expects:

```markdown
---
phase: 142
phase_name: discussion-audit-trail
date: 2026-04-07
decision_count: 5
next_step: /gsd-plan-phase 142
tags: [backend, library]
---

## Key Decisions

- D-01: Use pure functions only, no fs imports in library
- D-02: Three usage modes for pd:audit
- D-03: AND logic for combined filters
```
