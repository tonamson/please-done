  <action>
Create BusinessLogicMapper class that detects state machines and identifies business logic flaws per D-01, D-02, D-03.

Implementation requirements per 115-RESEARCH.md:

1. **State Machine Detection (D-01):**
   - Detect React useState: pattern `const [state, setState] = useState(initial)`
   - Detect Redux reducers: function with `state` param + `action` param + switch on `action.type`
   - Detect XState machines: `createMachine()` or `Machine()` calls
   - Detect Vuex/Pinia: `defineStore()` calls or store objects with state/mutations/actions
   - Detect manual state enums: variables named `*State` or `*Status` with object/array literals

2. **State Machine Structure:**
   ```javascript
   {
     name: string,
     type: 'react-useState' | 'redux-reducer' | 'xstate-machine' | 'vuex-store' | 'pinia-store' | 'manual-enum',
     states: [{ name, isInitial, isFinal }],
     transitions: [{ from, to, event, type }],
     location: { file, line, column }
   }
   ```

3. **Business Logic Flaw Detection (D-03):**
   - Missing auth checks: sensitive operations without auth-check step
   - Race condition patterns: multiple async operations without proper sequencing
   - State validation gaps: transitions without validation
   - Workflow bypass opportunities: routes accessible out of sequence

4. **Risk Scoring per D-17 through D-21:**
   - Critical: Business workflow bypass (skip payment, escalate privilege) per D-18
   - High: Missing auth in state transitions per D-19
   - Medium: Workflow timing issues per D-20
   - Low: State machine complexity warnings per D-21

5. **Mermaid State Diagram Generation (D-04):**
   ```javascript
   generateMermaid(stateMachine) {
     // Output: stateDiagram-v2 format
     // [*] --> InitialState
     // StateA --> StateB : eventName
     // note right of StateA : Risk annotation
   }
   ```

Use @babel/parser and @babel/traverse (already in package.json). Follow source-mapper.js patterns for AST traversal. Cache results using recon-cache.js pattern.

Dependencies: recon-cache.js (for caching), @babel/parser, @babel/traverse.

3. **Entry Point Identification:**
