Create TaintEngine class implementing multi-hop taint analysis per D-06, D-07, D-08, D-10.

Implementation requirements per 115-RESEARCH.md:

1. **Core Architecture:**
   ```javascript
   class TaintEngine {
     constructor(options = {}) {
       this.maxDepth = options.maxDepth || 5; // per D-10
       this.taintMap = new Map(); // variable -> { source, hops, path }
       this.worklist = []; // BFS worklist for fixed-point iteration
     }
   }
   ```

2. **Source Categories (D-07):**
   - URL parameters: `req.query`, `req.params`
   - HTTP body: `req.body`
   - Headers: `req.headers`
   - Cookies: `req.cookies`
   - File uploads: `req.files`
   - WebSocket messages
   - GraphQL inputs

3. **Sink Categories (D-08):**
   - SQL queries: `db.query()`, `sequelize.query()`, raw SQL
   - Shell commands: `exec()`, `spawn()`, `child_process`
   - eval/Function: `eval()`, `new Function()`, `setTimeout(string)`
   - File system: `fs.*` with tainted paths
   - HTTP responses: `res.send()` with unsanitized content (XSS)
   - DOM manipulation: `innerHTML`, `document.write()`

4. **Multi-Hop Propagation (D-10):**
   Use worklist algorithm for fixed-point iteration:
   ```javascript
   async analyzeTaintFlow(ast, sources, sinks) {
     // Initialize with sources
     for (const source of sources) {
       this.markTainted(source.variable, source, 0);
     }
     // Worklist iteration
     while (this.worklist.length > 0) {
       const task = this.worklist.shift();
       this.propagateTaint(task, ast);
     }
     return this.findTaintedSinks(sinks);
   }
   ```

5. **Taint Propagation Patterns:**
   - Assignment: `tainted = x` -> `y = tainted` marks y tainted
   - Property access: `tainted.prop` -> result is tainted
   - Function calls: `fn(tainted)` -> check if sanitizer, else result tainted
   - Destructuring: `const { x } = tainted` -> x is tainted
   - Spread: `const y = { ...tainted }` -> y is tainted
   - Array element: `array[0] = tainted` -> array is tainted

6. **Scope-Aware Tracking:**
   Use Babel's scope API: `path.scope.getBinding(identifier)`
   Track scope chain for inter-procedural flows
   Handle variable shadowing correctly

7. **Depth Limiting:**
   Stop propagation at 5 hops per D-10
   Track hop count in taint metadata

8. **Integration with Phase 113:**
   ```javascript
   async analyze(filePath, sourceInfo) {
     // sourceInfo from source-mapper.js
     const sources = sourceInfo.sources;
     const sinks = sourceInfo.sinks;
     // Run taint analysis
   }
   ```

Dependencies: @babel/parser, @babel/traverse, recon-cache.js.

Create DataFlowGraph class that generates DOT format data flow graphs per D-11.

Implementation requirements per 115-RESEARCH.md:

1. **DOT Graph Generation:**
   Generate DOT format as plain strings (no external dependency needed):
   ```javascript
   generateDOT() {
     const lines = ['digraph TaintGraph {'];
     lines.push('  rankdir=LR;');
     lines.push('  node [fontname="Arial", fontsize=12];');

     // Add node definitions with styling
     for (const [id, node] of this.nodes) {
       const style = this.getNodeStyle(node.type);
       lines.push(`  "${id}" [label="${node.label}", ${style}];`);
     }

     // Add edges
     for (const edge of this.edges) {
       lines.push(`  "${edge.from}" -> "${edge.to}";`);
     }

     lines.push('}');
     return lines.join('\n');
   }
   ```

2. **Node Types and Styling:**
   ```javascript
   getNodeStyle(type) {
     const styles = {
       source: 'shape=ellipse, fillcolor=lightcoral, style=filled',
       transform: 'shape=box, fillcolor=lightblue, style=filled',
       sink: 'shape=diamond, fillcolor=lightgoldenrod1, style=filled',
       sanitizer: 'shape=hexagon, fillcolor=lightgreen, style=filled'
     };
     return styles[type] || 'shape=box';
   }
   ```

3. **Graph Construction from Taint Results:**
   ```javascript
   buildFromTaintResults(taintFlows) {
     for (const flow of taintFlows) {
       const sourceId = this.addNode(
         `${flow.source.type}\n${flow.source.code.substring(0, 30)}`,
         'source'
       );

       let prevId = sourceId;

       // Add transformations
       for (const transform of flow.path || []) {
         const transformId = this.addNode(
           `${transform.type}\n${transform.variable || 'unnamed'}`,
           'transform'
         );
         this.addEdge(prevId, transformId, transform.type);
         prevId = transformId;
       }

       // Add sink
       for (const sink of flow.sinks) {
         const sinkId = this.addNode(
           `${sink.type}\n${sink.code.substring(0, 30)}`,
           'sink'
         );
         this.addEdge(prevId, sinkId);
       }
     }

     return this.generateDOT();
   }
   ```

4. **DOT Label Escaping:**
   ```javascript
   escapeLabel(label) {
     return label
       .replace(/"/g, '\\"')
       .replace(/\n/g, '\\n')
       .substring(0, 50); // Truncate long labels
   }
   ```

5. **Mermaid Conversion (Optional):**
   DOT graphs can be converted to Mermaid for display:
   ```javascript
   toMermaid(dotContent) {
     // Convert DOT to Mermaid flowchart syntax
     // Useful for embedding in markdown reports
   }
   ```

6. **API Design:**

     addNode(label, type, metadata) { ... }
     addEdge(from, to, label) { ... }
