Create a `pd:stats` skill that reads `.planning/` files and git history to display comprehensive project statistics: phase counts, plan counts, requirement counts, milestone progress, timeline, LOC counts, and file counts. Purpose: Provide users with a quick, comprehensive project health overview from the command line, eliminating manual file inspection to understand project scope and progress.

Output: Three new files — `bin/lib/stats-collector.js` (pure-function library), `test/stats-collector.test.js` (unit tests), `commands/pd/stats.md` (skill definition).

    - countProjectFiles: given rootDir string (project root path), counts .js files in bin/, commands/, workflows/, templates/, references/, test/ directories recursively; excludes node_modules/, .git/, FastCode/, .planning/milestones/, test/snapshots/; returns { totalFiles, totalLoc, breakdown: [{dir, files, loc}] }
    - countProjectFiles: returns { totalFiles: 0, totalLoc: 0, breakdown: [] } when directories don't exist
    - extractTimeline: given gitLog string (output of `git log --oneline`) and milestones array (from parseRoadmapPhases), returns [{milestone, startDate, completionDate, phases, plans}] — parses commit messages for milestone identifiers (e.g., "v12.1", "complete-milestone"); falls back to milestones array dates when git log is empty
    - extractTimeline: returns [] when both inputs are empty
    - formatStatsTable: given collected stats object { state, phases, requirements, files, timeline }, returns formatted string with boxed unicode borders (using ╔═╗║╚╝ characters matching dashboard-renderer.js banner style) with sections: Overview, Milestone Progress, Timeline, File Stats
    - formatStatsJson: given same stats object, returns JSON string (JSON.stringify with 2-space indent)
  </behavior>
  <action>
    **Create `bin/lib/stats-collector.js`** using CommonJS (`module.exports = { ... }`), `'use strict'`, with these pure functions. All functions receive content as arguments — NO file I/O in this module.

    1. `parseRoadmapPhases(content)` — Parse ROADMAP.md string. Extract:
       - Phase entries from `### Phase N:` headings → collect name, number, status (from `[x]` vs `[ ]` prefix in the `- [ ] **Phase N:**` lines in the Phases section)
       - Completed milestones from `✅ ** vX.Y ...**` lines in the Milestones section → extract milestone name, date if present
       - Progress table entries from `| Phase | Plans Complete | Status | Completed |` table rows
       - Return `{ totalPhases, completedPhases, phases: [{name, status}], milestones: [{name, status, date}], progressTable: [{phase, plansComplete, status, completed}] }`

    2. `parseStateProgress(content)` — Parse STATE.md string. Extract from YAML frontmatter:
       - `milestone`, `milestone_name`, `phase`, `status`
       - `progress.total_phases`, `progress.completed_phases`, `progress.total_plans`, `progress.completed_plans`
       - Use regex extraction pattern from dashboard-renderer.js: `^key:\s*(.+)$` for flat fields, nested `progress:` section parsing for sub-fields
       - Return `{ milestone, milestoneName, phase, status, progress: { total_phases, completed_phases, total_plans, completed_plans } }`

    3. `parseRequirements(content)` — Parse REQUIREMENTS.md string. Count checkbox items:
       - Count `- [x]` lines as completed
       - Count `- [ ]` lines as active (incomplete)
       - Total = completed + active
       - Return `{ total, completed, active }`

    4. `countProjectFiles(rootDir)` — This is the ONLY function that does file I/O (reads directory structure). Use `fs.readdirSync` with `withFileTypes: true` for recursive directory walking. Count:
       - Directories to scan: `bin/`, `commands/`, `workflows/`, `templates/`, `references/`, `test/`
       - Only `.js` files
       - Line count per file via `fs.readFileSync(path, 'utf8').split('\n').length`
       - Exclude paths containing: `node_modules`, `.git`, `FastCode`, `.planning/milestones`, `test/snapshots`
       - Return `{ totalFiles, totalLoc, breakdown: [{dir, files, loc}] }` where `dir` is the top-level directory name (e.g., "bin", "test")

    5. `extractTimeline(gitLog, milestones)` — Parse git log for milestone dates:
       - Parse `gitLog` (string from `git log --oneline`) for commits containing milestone identifiers (pattern: `v\d+\.\d+` or `complete-milestone`)
       - Map milestone names from the `milestones` array to their git commit dates
       - For milestones not found in git log, use dates from the milestones array (fallback per D-18)
       - Return `[{milestone, startDate, completionDate, phases, plans}]`

    6. `formatStatsTable(stats)` — Render stats as boxed table with unicode borders:
       - Use ╔═╗║╚╝ characters consistent with `utils.js` banner style
       - Sections: Overview (milestone, phase, status, phases/plans/requirements counts), Milestone Progress (current milestone progress bar), Timeline (table: Milestone | Start | Completed | Phases | Plans), File Stats (table: Directory | Files | LOC, plus totals row)
       - Use `padRight` helper (copy from dashboard-renderer.js pattern) for column alignment
       - Return complete formatted string

    7. `formatStatsJson(stats)` — Return `JSON.stringify(stats, null, 2)`
