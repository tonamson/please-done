**Exported Functions:**
1. `parseUrl(url)` - Parse URL into components (protocol, host, port, path, query, fragment). Returns `{ protocol, host, port, path, query, params: [{key, value}], fragment }`

2. `parseQueryString(query)` - Parse query string into key-value pairs. Returns `[{key, value, decoded}]`

3. `parseHeaders(rawHeaders)` - Parse HTTP headers into structured object. Returns `{ headers: [{name, value, risk}], suspicious: [] }`. Flag suspicious headers like X-Forwarded-For, X-Real-IP, CF-Connecting-IP

4. `enumeratePaths(basePath, depth)` - Enumerate common paths from base. Use wordlist: `/api`, `/admin`, `/debug`, `/test`, `/backup`, `/config`, `/v1`, `/v2`, `/internal`, `/private`. Returns `[{path, description, risk}]`

5. `detectEndpoints(content)` - Detect API endpoints from content (HTML, JS). Regex for `fetch(`, `axios.`, `/api/`, endpoint patterns. Returns `{endpoints: [{url, method, line}], count}`

6. `analyzeTarget(target)` - Main analysis: parse URL, headers, enumerate paths, detect endpoints. Returns `{ url, headers, paths, endpoints, summary: { pathCount, endpointCount, headerRisk } }`

**MITRE ATT&CK mappings:**
- T1593.002: Search engine discovers target subdomains
- T1596.003: Certificate transparency reveals subdomains

**Pattern:** Follow existing library pattern from payloads.js and token-analyzer.js - JSDoc comments, "use strict", structured results, MITRE references in docstring.
