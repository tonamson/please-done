  <action>
Create bin/lib/token-analyzer.js with the TokenAnalyzer class:

1. **Constructor**: Accept { cache } options, initialize ReconCache

2. **JWT Analysis (analyzeJwt(token))**:
   - Parse JWT header and payload (base64 decode, JSON.parse)
   - Detect alg:none vulnerability: if header.alg === 'none' or missing algorithm, flag as CRITICAL
   - Detect weak secrets: Check for common short keys ('secret', 'password', '1234', 'admin', <8 chars) via commonWeakSecrets list
   - Detect missing algorithm verification: Flag when jwt.verify() called without explicit algorithms option
   - Report JWT expiration: Check exp claim, flag if expired or within 24h of expiry
   - Calculate token age: Estimate from iat claim if present
   - Return: { type: 'jwt', vulnerabilities: [], severity: 'CRITICAL|HIGH|MEDIUM|LOW', details: {} }

3. **Patterns to detect**:
   - `jwt.sign(` with weak secret
   - `jwt.verify(token,` without algorithms option
   - JWT tokens in source: /eyJ[A-Za-z0-9-_]+\.eyJ[A-Za-z0-9-_]+/g

4. **Export**: module.exports = { TokenAnalyzer };
  </action>

<task type="auto">
  <name>Task 2: Implement session cookie security analysis (TOKEN-02)</name>
  <files>bin/lib/token-analyzer.js</files>
  <action>
Extend bin/lib/token-analyzer.js with session cookie analysis:

1. **Cookie Security Analysis (analyzeCookie(cookieString))**:
   - Parse cookie string into name=value pairs
   - Check security flags:
     - HttpOnly: Missing = HIGH risk (XSS can access)
     - Secure: Missing = HIGH risk (sent over HTTP)
     - SameSite: Missing or 'None' without Secure = MEDIUM/HIGH
     - Priority: 'Low'/'Medium'/'High' - informational
   - Calculate entropy for session ID values:
     - Use Shannon entropy formula: -sum((count/len)*log2(count/len))
     - Low entropy (<3.0) = predictable = CRITICAL
     - Medium entropy (3.0-4.0) = possible = HIGH
   - Detect predictable patterns: Check for sequential IDs, timestamps, user info in value

2. **Cookie Extraction**:
   - Detect set-cookie headers in source code
   - Detect document.cookie assignments
   - Detect cookie-parser usage patterns

3. **Return format**:
   ```javascript
   {
     type: 'session-cookie',
     name: string,
     value: string,
     flags: { httpOnly, secure, sameSite, priority },
     entropy: number,
     predictability: 'CRITICAL|HIGH|MEDIUM|LOW',
     vulnerabilities: [],
     severity: 'CRITICAL|HIGH|MEDIUM|LOW'
   }
   ```
  </action>
  <verify>
    <automated>node -e "const { TokenAnalyzer } = require('./bin/lib/token-analyzer'); const ta = new TokenAnalyzer(); const result = ta.analyzeCookie('session=abc123; HttpOnly; Secure; SameSite=Strict'); console.log('Cookie analyzed:', result.type);"</automated>
  </verify>
