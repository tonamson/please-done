## Task 1: Create Service Discovery Library

**Goal:** Implement ServiceDiscovery class for technology stack fingerprinting

**Steps:**
1. Create `bin/lib/service-discovery.js` with:
   - Framework detection (Express, Fastify, NestJS, Next.js, Koa, Hapi)
   - Version extraction from package.json and package-lock.json
   - Dependency mapping (databases, ORMs, auth, external services)
   - Vulnerability checking via osv.dev API
   - Integration with ReconCache

2. Implement methods:
   - `analyzeDependencies(projectPath)` - Parse package.json and identify all dependencies
   - `detectFramework()` - Detect primary framework and version
   - `getTechStack()` - Return complete technology stack mapping
   - `checkVulnerabilities()` - Query osv.dev for known CVEs in dependencies
   - `isOutdated(packageName, currentVersion)` - Check if version is outdated

3. Framework detection patterns:
   ```javascript
   // Check imports and package.json dependencies
   express: 'express' in dependencies
   fastify: 'fastify' in dependencies
   nestjs: '@nestjs/core' in dependencies
   nextjs: 'next' in dependencies
   ```

4. Vulnerability checking:
   - Query osv.dev API: `https://api.osv.dev/v1/query`
   - Request format: `{ package: { name, ecosystem }, version }`
   - Cache results in ReconCache with 24hr TTL
   - Rate limiting: 50 requests/minute

5. Output format:
   ```javascript
   {
     framework: {
       name: 'Express',
       version: '4.18.2',
       confidence: 'high'
     },
     database: {
       name: 'PostgreSQL',
       orm: 'Sequelize',
       version: '6.32.1'
     },
     auth: {
       strategy: 'JWT',
       library: 'jsonwebtoken',
       version: '9.0.0'
     },
     dependencies: [
       { name: 'lodash', version: '4.17.21', vulnerabilities: [] },
       { name: 'axios', version: '0.26.0', vulnerabilities: ['CVE-2021-...'] }
     ],
     outdated: [
       { name: 'axios', current: '0.26.0', latest: '1.6.0' }
     ]
   }
   ```

**Commit:** `feat(phase-113): implement service discovery for tech stack fingerprinting`

## Task 2: Create Recon Aggregator

**Goal:** Implement ReconAggregator that combines all reconnaissance data

**Steps:**
1. Create `bin/lib/recon-aggregator.js` with:
   - Integration with SourceMapper (from 113-01)
   - Integration with TargetEnumerator (from 113-01)
   - Integration with ServiceDiscovery (from this plan)
   - Unified report generation
   - Risk scoring algorithm

2. Implement methods:
   - `runFullRecon(projectPath)` - Run all reconnaissance phases
   - `getSourceMap()` - Get untrusted data sources
   - `getTargetMap()` - Get discovered endpoints
   - `getTechStack()` - Get technology inventory
   - `getRiskReport()` - Generate prioritized risk findings
   - `exportToJson()` - Export full report as JSON

3. Risk scoring algorithm:
   ```javascript
   // Risk factors
   - Exposed sensitive routes without auth: 10 points
   - Outdated dependencies with CVEs: 8 points
   - SQL injection potential: 9 points
   - Command injection potential: 9 points
   - XSS injection potential: 7 points
   - Missing input validation: 5 points
   - Weak authentication: 6 points
   ```

4. Report format:
   ```javascript
   {
     summary: {
       sourcesFound: 15,
       routesDiscovered: 42,
       vulnerabilities: 3,
       outdatedDependencies: 5,
       overallRisk: 'high' // high/medium/low
     },
     sources: [...],      // From SourceMapper
     targets: [...],      // From TargetEnumerator
     techStack: {...},    // From ServiceDiscovery
     risks: [...],        // Aggregated risk findings
     recommendations: [...] // Prioritized remediation steps
   }
   ```

**Commit:** `feat(phase-113): implement recon aggregator for unified reporting`
